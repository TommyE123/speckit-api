# Implementation Plan: JSONPlaceholder Core Data Layer

**Branch**: `001-jsonplaceholder-core` | **Date**: 2026-05-23 | **Spec**: [spec.md](spec.md)

**Input**: Feature specification from `specs/001-jsonplaceholder-core/spec.md`

---

## Summary

Build the .NET 10 core data layer for a JSONPlaceholder integration: a typed HTTP client that fetches albums and photos from the external API, DTOs that capture the external JSON shapes, internal domain models that represent those entities cleanly, and a service layer that combines and filters the data in memory. Full unit test coverage is required; no API endpoints are in scope for this feature.

---

## Technical Context

**Language/Version**: C# 13 / .NET 10

**Primary Dependencies**:
- `Microsoft.Extensions.Http` — `IHttpClientFactory` + typed client registration
- `Microsoft.Extensions.Http.Polly` — retry and timeout policies via Polly
- `System.Text.Json` — serialization (built-in, no additional package)
- `xunit` + `xunit.runner.visualstudio` — test runner
- `Moq` — mocking `IJsonPlaceholderClient` in service unit tests

**Storage**: N/A — data is fetched from the external JSONPlaceholder API and held in memory per request. No persistence layer.

**Testing**: xUnit + Moq. `MockHttpMessageHandler` (hand-rolled `DelegatingHandler` stub) for `JsonPlaceholderClient` unit tests.

**Target Platform**: .NET 10 — cross-platform (Linux/macOS/Windows). CI runs on the default GitHub-hosted runner.

**Project Type**: Web service (foundational library layer; HTTP host and endpoints deferred to feature 002).

**Performance Goals**: Fetch + combine 100 albums and 5,000 photos in under 2 seconds on a standard developer machine with a good network connection. Unit tests complete in under 5 seconds with no live network dependency (SC-004).

**Constraints**:
- No live network calls during `dotnet test` — all external HTTP is mocked.
- `System.Text.Json` only; Newtonsoft.Json is prohibited.
- Base URL must not be hardcoded; must come from `appsettings.json`.

**Scale/Scope**: Single bounded feature — 5 source files, 2 test files, ~300–400 lines of production code, ~200–300 lines of test code.

## Architecture Decisions

### A1 — Typed HTTP Client as Interface + Implementation

`IJsonPlaceholderClient` defines two async methods:

```csharp
Task<IReadOnlyList<AlbumDto>> GetAlbumsAsync(CancellationToken cancellationToken = default);
Task<IReadOnlyList<PhotoDto>> GetPhotosAsync(CancellationToken cancellationToken = default);
```

`JsonPlaceholderClient` implements this via an injected `HttpClient` (provided by `IHttpClientFactory`). The interface makes the client fully mockable by `AlbumService` unit tests.

### A2 — Polly Policies at Registration

Registered in DI:
- **Retry**: `AddTransientHttpErrorPolicy` — 3 retries with exponential backoff (1s, 2s, 4s). Handles transient 5xx and network errors.
- **Timeout**: `AddPolicyHandler(Policy.TimeoutAsync<HttpResponseMessage>(10))` — 10-second per-request timeout.

Policies are applied as handler pipeline via `AddHttpClient<IJsonPlaceholderClient, JsonPlaceholderClient>().AddPolicyHandler(...).AddPolicyHandler(...)`.

### A3 — In-Memory Combine and Filter

`AlbumService.GetAlbumsWithPhotosAsync()`:
1. Fetch `IReadOnlyList<AlbumDto>` and `IReadOnlyList<PhotoDto>` concurrently via `Task.WhenAll`.
2. Build a `Dictionary<int, List<Photo>>` keyed by `albumId` from the mapped photos.
3. Map each `AlbumDto` to `Album` and construct `AlbumWithPhotos` from the lookup.
4. Return `IReadOnlyList<AlbumWithPhotos>`.

`AlbumService.GetAlbumsWithPhotosByUserAsync(int userId)`:
1. Call `GetAlbumsWithPhotosAsync()`.
2. Filter the result with `.Where(a => a.Album.UserId == userId)`.
3. Return filtered list (empty list when no match — FR-006).

Concurrent fetch is safe: both `/albums` and `/photos` are read-only GET requests.

### A4 — DTO → Domain Mapping in Service

Mapping is inline in `AlbumService` (no AutoMapper). The only non-trivial rename is `PhotoDto.Url` → `Photo.ImageUrl`. This is documented in `data-model.md` and explicit in code.

### A5 — Options Pattern for Configuration

`JsonPlaceholderOptions` POCO bound from `appsettings.json`:

```json
{
  "JsonPlaceholderOptions": {
    "BaseUrl": "https://jsonplaceholder.typicode.com"
  }
}
```

Injected via `IOptions<JsonPlaceholderOptions>` in `JsonPlaceholderClient`. Overridable via environment variables in CI and test environments.

### Phase 3 — Implementation (Green)

**Goal**: Implement production code to make all tests pass. Follow TDD — implement only what is needed to pass the failing tests.

#### 3a — `JsonPlaceholderClient`

```csharp
public sealed class JsonPlaceholderClient : IJsonPlaceholderClient
{
    private readonly HttpClient _httpClient;

    public JsonPlaceholderClient(HttpClient httpClient) => _httpClient = httpClient;

    public async Task<IReadOnlyList<AlbumDto>> GetAlbumsAsync(CancellationToken ct = default)
    {
        var result = await _httpClient.GetFromJsonAsync<List<AlbumDto>>("/albums", ct);
        return result ?? [];
    }

    public async Task<IReadOnlyList<PhotoDto>> GetPhotosAsync(CancellationToken ct = default)
    {
        var result = await _httpClient.GetFromJsonAsync<List<PhotoDto>>("/photos", ct);
        return result ?? [];
    }
}
```

Key points:
- Uses `GetFromJsonAsync<T>()` (from `System.Net.Http.Json`) for concise deserialization.
- Returns empty list on null response (defensive; JSONPlaceholder always returns a non-null array).
- Polly retry and timeout are applied at the registration layer, not in the client body.

#### 3b — `AlbumService`

```csharp
public sealed class AlbumService : IAlbumService
{
    private readonly IJsonPlaceholderClient _client;

    public AlbumService(IJsonPlaceholderClient client) => _client = client;

    public async Task<IReadOnlyList<AlbumWithPhotos>> GetAlbumsWithPhotosAsync(CancellationToken ct = default)
    {
        var (albumDtos, photoDtos) = await FetchBothAsync(ct);

        var photosByAlbumId = photoDtos
            .Select(MapPhoto)
            .GroupBy(p => p.AlbumId)
            .ToDictionary(g => g.Key, g => (IReadOnlyList<Photo>)g.ToList());

        return albumDtos
            .Select(a => new AlbumWithPhotos(
                MapAlbum(a),
                photosByAlbumId.GetValueOrDefault(a.Id, [])))
            .ToList();
    }

    public async Task<IReadOnlyList<AlbumWithPhotos>> GetAlbumsWithPhotosByUserAsync(
        int userId, CancellationToken ct = default)
    {
        var all = await GetAlbumsWithPhotosAsync(ct);
        return all.Where(a => a.Album.UserId == userId).ToList();
    }

    private async Task<(IReadOnlyList<AlbumDto>, IReadOnlyList<PhotoDto>)> FetchBothAsync(CancellationToken ct)
    {
        var albumsTask = _client.GetAlbumsAsync(ct);
        var photosTask = _client.GetPhotosAsync(ct);
        await Task.WhenAll(albumsTask, photosTask);
        return (await albumsTask, await photosTask);
    }

    private static Album MapAlbum(AlbumDto dto) => new(dto.Id, dto.UserId, dto.Title);

    private static Photo MapPhoto(PhotoDto dto) => new(dto.Id, dto.AlbumId, dto.Title, dto.Url, dto.ThumbnailUrl);
}
```

**Exit criteria**: All tests from Phase 2 pass. Zero test failures.

#### 4b — DI Extension Method

Create `ServiceCollectionExtensions.cs` in `src/SpecKitApi/` (or inside an `Extensions/` subfolder):

```csharp
public static class ServiceCollectionExtensions
{
    public static IServiceCollection AddJsonPlaceholderServices(
        this IServiceCollection services, IConfiguration configuration)
    {
        services.Configure<JsonPlaceholderOptions>(
            configuration.GetSection(JsonPlaceholderOptions.SectionName));

        services.AddHttpClient<IJsonPlaceholderClient, JsonPlaceholderClient>((sp, client) =>
        {
            var opts = sp.GetRequiredService<IOptions<JsonPlaceholderOptions>>().Value;
            client.BaseAddress = new Uri(opts.BaseUrl);
        })
        .AddTransientHttpErrorPolicy(p =>
            p.WaitAndRetryAsync(3, attempt => TimeSpan.FromSeconds(Math.Pow(2, attempt - 1))))
        .AddPolicyHandler(Policy.TimeoutAsync<HttpResponseMessage>(10));

        services.AddScoped<IAlbumService, AlbumService>();

        return services;
    }
}
```

## Interface Contracts (Internal)

These are not published OpenAPI contracts (no endpoints exist yet), but serve as the authoritative internal contracts between layers.

### `IJsonPlaceholderClient`

```csharp
/// <summary>Fetches raw data from the JSONPlaceholder external API.</summary>
public interface IJsonPlaceholderClient
{
    /// <summary>Returns all albums from /albums.</summary>
    Task<IReadOnlyList<AlbumDto>> GetAlbumsAsync(CancellationToken cancellationToken = default);

    /// <summary>Returns all photos from /photos.</summary>
    Task<IReadOnlyList<PhotoDto>> GetPhotosAsync(CancellationToken cancellationToken = default);
}
```

### `IAlbumService`

```csharp
/// <summary>Combines and filters album and photo data from the JSONPlaceholder source.</summary>
public interface IAlbumService
{
    /// <summary>
    /// Returns all albums, each containing its associated photos.
    /// </summary>
    Task<IReadOnlyList<AlbumWithPhotos>> GetAlbumsWithPhotosAsync(
        CancellationToken cancellationToken = default);

    /// <summary>
    /// Returns albums owned by the specified user, each containing its associated photos.
    /// Returns an empty list when the user owns no albums.
    /// </summary>
    Task<IReadOnlyList<AlbumWithPhotos>> GetAlbumsWithPhotosByUserAsync(
        int userId,
        CancellationToken cancellationToken = default);
}
```
